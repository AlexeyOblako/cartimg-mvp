# Баг 1: Отсутствие валидации имени и телефона

## Симптом
При бронировании заезда можно указать невалидные данные: имя "1" (одна цифра) и телефон "1" (одна цифра). Система принимает эти данные и создаёт бронирование.

## Требование
- Имя должно содержать только буквы (минимум 2 символа)
- Телефон должен содержать ровно 11 цифр (формат: 88005553312)

## Воспроизведение
1. Нажать "Забронировать" на любом заезде
2. В поле "Имя" ввести: 1
3. В поле "Телефон" ввести: 1
4. Ввести количество картов: 1
5. Нажать "Забронировать"

Фактический результат: Бронирование создано
Ожидаемый результат: Ошибка валидации

## Исправление
Добавлена валидация на бэкенде (backend/src/routes/bookings.js) и фронтенде (frontend/src/components/BookingForm.jsx):
- Проверка имени по регулярке: /^[a-zA-Zа-яА-ЯёЁ\s\-]{2,}$/
- Проверка телефона по регулярке: /^\d{11}$/

## Проверка вручную
1. Повторить шаги воспроизведения
2. Результат: Появляется ошибка "Имя должно содержать только буквы..."

cURL для проверки:
```bash
curl -X POST http://localhost:3001/api/bookings -H "Content-Type: application/json" -d '{"session_id":1,"customer_name":"1","customer_phone":"1","karts_count":1}'
```

Ожидаемый ответ:
```json
{"error": "Имя должно содержать только буквы (минимум 2 символа). Пример: Иван Иванов"}
```

ТРЕБОВАНИЯ К ВАЛИДАЦИИ:
- Имя (customer_name): только буквы (русские и английские), пробелы, дефисы. Минимум 2 символа. Регулярка: /^[a-zA-Zа-яА-ЯёЁ\s\-]{2,}$/
- Телефон (customer_phone): только цифры (11 штук). Без плюса, скобок, пробелов, дефисов. Регулярка: /^\d{11}$/
