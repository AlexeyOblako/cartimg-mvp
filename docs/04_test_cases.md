# Тест-кейсы: POST /api/bookings

## Тест 1: Успешное бронирование

**Описание:** Клиент бронирует 2 карта на существующий заезд, где достаточно свободных мест.

**Шаги:**
1. Отправить POST-запрос на `/api/bookings` с корректными данными
2. Проверить статус ответа 201
3. Проверить тело ответа

**Ожидаемый результат:** Бронь создана. В ответе: `id`, `session_id`, `customer_name`, `status: "active"`, `message: "Бронь подтверждена"`. `available_karts` в сессии уменьшился на `karts_count`.

**cURL для проверки:**
```bash
curl -X POST http://localhost:3001/api/bookings \
  -H "Content-Type: application/json" \
  -d '{"session_id":1,"customer_name":"Иван","customer_phone":"+79991234567","karts_count":2}'
```

Ожидаемый ответ:
```json
{
  "id": 1,
  "session_id": 1,
  "customer_name": "Иван",
  "status": "active",
  "message": "Бронь подтверждена"
}
```

## Тест 2: Бронирование несуществующего заезда

**Описание:** Клиент пытается забронировать карты на заезд, которого нет в БД.

**Шаги:**
1. Отправить POST-запрос на `/api/bookings` с `session_id`, которого не существует (например, 999)
2. Проверить статус ответа 404

**Ожидаемый результат:** Бронь не создана. Сервер возвращает 404 и сообщение `"Заезд не найден"`.

**cURL для проверки:**
```bash
curl -X POST http://localhost:3001/api/bookings \
  -H "Content-Type: application/json" \
  -d '{"session_id":999,"customer_name":"Иван","customer_phone":"+79991234567","karts_count":1}'
```

Ожидаемый ответ:
```json
{ "error": "Заезд не найден" }
```

## Тест 3: Бронирование при отсутствии мест

**Описание:** Клиент пытается забронировать больше картов, чем доступно.

**Шаги:**
1. Выполнить успешное бронирование, чтобы уменьшить `available_karts` (если нужно)
2. Отправить POST-запрос с `karts_count`, превышающим оставшиеся свободные карты
3. Проверить статус ответа 400

**Ожидаемый результат:** Бронь не создана. Сервер возвращает 400 и сообщение `"Нет свободных картов. Доступно: N, запрошено: M"`.

**cURL для проверки:**
```bash
curl -X POST http://localhost:3001/api/bookings \
  -H "Content-Type: application/json" \
  -d '{"session_id":4,"customer_name":"Иван","customer_phone":"+79991234567","karts_count":99}'
```

Ожидаемый ответ:
```json
{ "error": "Нет свободных картов. Доступно: 6, запрошено: 99" }
```

## Тест 4: Бронирование с пустыми полями

## Тест 5: Бронирование с karts_count = 0

**Описание:** Клиент отправляет запрос с нулевым количеством картов.

**Шаги:**
1. Отправить POST-запрос с `karts_count = 0`
2. Проверить статус ответа

**Ожидаемый результат:** Зависит от бизнес-логики. При `karts_count = 0` условие `session.available_karts < karts_count` ложно (0 < 0 → false), поэтому бронь **создастся** (если сервер не добавит дополнительную проверку `karts_count > 0`). В текущей реализации статус 201.

**cURL для проверки:**
```bash
curl -X POST http://localhost:3001/api/bookings \
  -H "Content-Type: application/json" \
  -d '{"session_id":1,"customer_name":"Иван","customer_phone":"+79991234567","karts_count":0}'
```

Ожидаемый ответ (текущая логика):
```json
{
  "id": 2,
  "session_id": 1,
  "customer_name": "Иван",
  "status": "active",
  "message": "Бронь подтверждена"
}
```

> **Рекомендация:** Добавить валидацию `karts_count > 0`, чтобы исключить брони с нулевым количеством картов.
